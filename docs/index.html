Reading the ring

The bright fill shows your % used. The thin white spoke marks where a fair pace would put you right now — calculated from time elapsed in the rolling window. If your fill has passed the spoke, you're consuming faster than evenly. If it's behind, you have headroom.

What the color says

Color is purely the gap between fill and spoke. Cyan when you're well behind the spoke, lime when on pace, soft periwinkle as you start to creep ahead, deep violet when you're well past it. No alarms — just at-a-glance answers.

Source of truth

claude-meter reads the same numbers Claude Code's own /usage panel displays, via a statusline hook that captures rate_limits.five_hour and rate_limits.seven_day. When no live data is available, the rings stay empty — no estimates, no guesses.

Where it lives

A 140×140 overlay pinned to the top-right of your rightmost screen via PyObjC. Stays above app windows on every macOS Space. Restartable via launchd so it survives logout.

Install

# clone, install, run — that's it
git clone https://github.com/CasterlyGit/claude-meter
cd claude-meter
python3.12 -m venv .venv && source .venv/bin/activate
pip install -e .
claude-meter # overlay appears top-right of your rightmost display
